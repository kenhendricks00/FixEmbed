# Spec: FixEmbed v1.3 First-Party Core

## Objective

Ship the first measurable step toward FixEmbed becoming the most reliable and polished Discord embed bot. Members should be able to paste one or more supported links into `/fix` and receive consistently branded FixEmbed URLs. X/Twitter embeds should normally be rendered by the FixEmbed Cloudflare Worker, while FxTwitter remains an emergency fallback only. Status output must report honest live probe results.

### Acceptance Criteria

- `/fix` converts every supported, non-suppressed link in the supplied text, preserving input order.
- Automatic messages, the message context command, and `/fix` use the same parser and display metadata.
- Invalid input receives a private, actionable response listing supported services.
- Native and already-fixed supported URLs normalize to one canonical original URL.
- The active X/Twitter handler fetches and renders tweet data through FixEmbed.
- X/Twitter falls back to FxTwitter only when the first-party fetch fails.
- Status responses distinguish first-party success, fallback operation, degradation, and outage without synthetic uptime history.
- Core platform support, embed quality, and `/fix` remain free.

## Tech Stack

- Discord bot: Python 3.12, discord.py, aiosqlite.
- Embed service: TypeScript, Hono, Cloudflare Workers.
- Tests: Python `unittest`; Node's assertion API; TypeScript compiler.
- Deployment: SparkedHost for the bot; Cloudflare Workers for `fixembed.app`.

## Commands

- Python compile: `python -m compileall -q main.py translations.py link_utils.py`
- Python tests: `python -m unittest discover -s tests -v`
- Service install: `cd service && npm ci`
- Service tests: `cd service && npm test`
- Service types: `cd service && npx tsc --noEmit`
- Production audit: `cd service && npm audit --omit=dev`
- Worker development: `cd service && npm run dev`
- Worker deployment: `cd service && npm run deploy`

## Project Structure

- `main.py` — Discord orchestration, commands, settings, delivery.
- `link_utils.py` — pure canonical URL parsing and presentation metadata.
- `tests/` — Python behavior tests for shared bot logic.
- `service/src/handlers/` — one handler per platform.
- `service/src/utils/` — embed, fetch, status, and site utilities.
- `service/tests/` — Worker parser, handler, and status tests.
- `docs/ideas/` — confirmed product direction.
- `docs/specs/` — release-scoped implementation contracts.

## Code Style

```python
result = parse_supported_links(message_content)
if not result.links:
    return unsupported_links_message(result.unsupported_urls)
```

Use pure functions for parsing and formatting, explicit result types at boundaries, descriptive behavior-based test names, and orchestration code that does not duplicate platform regexes.

## Testing Strategy

- Small tests cover URL normalization, suppression, ordering, display labels, fallback classification, and status assessment.
- Handler tests use stubbed fetch responses for first-party success, deleted/private posts, media, and upstream failure.
- The full Python and TypeScript suites run after each vertical slice.
- Live verification uses a known public X/Twitter post through `fixembed.app` and a SparkedHost bot restart.

## Boundaries

- Always: keep core capabilities free; validate URL hosts; preserve emergency fallbacks; update changelog and release metadata; test before commits.
- Ask first: new paid entitlements, destructive database migrations, new external vendors, or Discord Developer Portal changes.
- Never: commit secrets; fetch arbitrary user-controlled hosts; publish invented uptime data; remove a fallback before the replacement is proven.

## Implementation Slices

1. Canonical bot URL engine and multi-link `/fix`.
2. First-party X/Twitter handler with explicit fallback result metadata.
3. Honest live status model and status UI copy.
4. Changelog/version metadata, full quality review, deployment, and live verification.

## Success Criteria

- All acceptance criteria are covered by automated tests.
- No production dependency vulnerabilities are reported by `npm audit --omit=dev`.
- A live X/Twitter URL returns a FixEmbed-branded embed without redirecting during normal operation.
- The SparkedHost bot starts all configured shards and synchronizes commands.
- The worktree is clean and release notes accurately describe the shipped behavior.
