# Domain docs

FixEmbed uses a single domain context.

## Before exploring

- Read `CONTEXT.md` at the repository root.
- Read relevant system-wide decisions under `docs/adr/`.
- If either location does not exist, proceed without creating placeholder documentation.

## Vocabulary

Use the canonical terms defined in `CONTEXT.md` in specifications, tickets, tests, and implementation discussions. Avoid synonyms explicitly listed there.

## Decisions

If proposed work contradicts an existing ADR, surface the conflict explicitly instead of silently overriding it.
